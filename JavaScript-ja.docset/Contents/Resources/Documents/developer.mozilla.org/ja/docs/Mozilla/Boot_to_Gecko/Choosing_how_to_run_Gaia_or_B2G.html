    Gaia もしくは B2G の実行方法を選択する

    どのような用途で使用するかで、Boot to Gecko もしくは Gaia ユーザインタフェースを検証する際に、よく考慮すべきオプションがあります。選択可能なのは以下のオプションですが、どれにも考えるべきメリットとデメリットがあり、他のものよりもフレキシブルに利用できるものもあります。

    Firefox で Gaia を実行する

    自分が何をしているのかわかっている場合は、Gaia リポジトリ を複製し、Firefox 15 以降で Gaia を開いて起動することができます。

    メリット

    • 何もビルドする必要がありませんし、セットアップに必要なものもほとんどありません
    • Firefox に組み込まれている開発者ツールをデバッグに使用できます

    デメリット

    • ビューポートはデバイスの画面サイズではなく、ブラウザウィンドウの大きさに基づきます
    • 多くのハードウェアインタフェースは動作しません
    • アプリケーションは、メイン UI タブではなく、個別の固定タブに起動します
    • ナイトリービルドを利用する場合、Firefox ナイトリービルドは安定しない可能性があります

    Firefox で Gaia を実行する理由

    ハードウェア上やシミュレータではなく、Firefox で Gaia を実行することを選択するのは、主に二つの理由があります。簡単に準備と実行ができ、Firefox の素晴らしい開発者ツールにアクセスできることです。これは Web アプリ開発者が アプリの UI や基本的な機能の初期テストを行うのに、また同様に Gaia ハッカーにとって、非常に大きなメリットといえるでしょう。

    注意 : アプリをリリースする前に、実際のハードウェア上でのテストが絶対に必要です！

    デスクトップで B2G を実行する

    Boot to Gecko シミュレータをビルドして、その上で Gaia を実行することが可能です。このソフトウェアは Firefox をベースにしていますが、デバイス上の Boot to Gecko エクスペリエンスと非常に良く似た振る舞いをします。現在 Mozilla は開発者向けに、このアプリケーションのナイトリービルドを提供しています。Firefox コードベースや C++ プロジェクトに精通している場合は、このアプリケーションをこれらの説明に従って自分でビルドすることができます。

    メリット

    • モバイルサイズのビューポートを利用できます
    • あらゆる点で実際のモバイルデバイスに似たエクスペリエンスです
    • （全てではありませんが）より多くのデバイス API が利用できます

    デメリット

    • C/C++ ビルド環境を実際にインストールする必要があります
    • Gecko とシミュレータを自分でビルドする必要があります
    • Firefox 開発者ツールは利用できません

    B2G シミュレータで実行する理由

    これは程よくバランスのとれたテスト及び開発のソリューションです。何かテストしたいと思った時に、毎回実際にスマートフォンのフラッシュメモリに書き込まなくても、デバイスに似た環境で、あなたのアプリやその他のコードがどんな感じなのか、確認するのに便利です。

    注意 : アプリをリリースする前に、実際のハードウェア上でのテストが絶対に必要です！

    シミュレータの種類

    いくつかの種類の Firefox OS シミュレータ を使用することができます。

     Firefox OS シミュレータ アドオン
    この拡張機能は、Firefox OS 上で動作するテストアプリケーションのための主要なツールであり、ほとんどのユーザーに推奨されるソリューションです。これには開発者ツールが含まれており、テスト環境にアプリケーションを追加することもサポートされます。 
    開発者デスクトップビルド
    これらはスタンドアロンのアプリケーションであるシミュレータを構築します。コアなFirefox OSの開発者が技術的な特徴を確認する事を支援します。
    ローカライザーデスクトップビルド
    ローカライザービルドは、ローカライズチームが Firefox OS と Firefox OSのアプリをテストするために役立ちます。

    モバイルデバイス上で B2G を実行する

    B2G もしくは Gaia の開発作業や、Web アプリを最も完璧にテストする方法は、実際のモバイルデバイスに、ビルドした Boot to Gecko をインストールすることです。これは最も面倒な作業でもあります。

    メリット

    • 完璧なモバイルデバイスエクスペリエンスが得られます
    • 全てのデバイス API が利用できます
    • 実際のデバイスの使用法でコードのパフォーマンスが体験できます

    デメリット

    • C/C++ ビルド環境を全てインストールする必要があります
    • Gecko と Gaia を自分でビルドする必要があります
    • B2G オペレーティングシステムをインストールする、互換性のあるモバイルデバイスが必要です
    • 現在インストールされているオペレーティングシステムがなんであろうと削除して、B2G をデバイスのフラッシュメモリに書き込む必要があります

    モバイルデバイスで B2G を実行する理由

    これは、明らかに、B2G もしくは Gaia 上で、コードや Web プロジェクトのテストをする最も正確なやり方です。実際のモバイルハードウェア上で実行することで、プロジェクトがきちんと実行されるか、きれいに見えるか、全てのデバイス API が正しく使用されているかを、確認することができます。付け加えるなら、どんなコードでもリリースする前には、常に実際のハードウェアでテストすべきです。これを行わないと、どんなまずい影響が起こる可能性があるのか予測するのは困難です。
