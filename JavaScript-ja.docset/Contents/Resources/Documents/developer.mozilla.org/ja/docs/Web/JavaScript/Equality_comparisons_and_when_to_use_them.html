    等価性の比較とその使いどころ

    In This Article
    1. === による厳格な等価性
    2. == による寛容な等価性
    3. 同一値の等価性

    JavaScript には 3 種類の値比較演算子があります。=== による厳格な等価性 (trict equality)、== による寛容な等価性 (loose equality)、Object.is (ECMAScript 6 の新機能) です。どの演算子を使用するかは、どのような比較を行いたいかに依存します。

    === による厳格な等価性

    厳格な等価性は、2 つの値が等しいかを比較します。比較対象の値はどちらも、比較する前に別の値へ暗黙的に変換されることはありません。値の型が異なる場合、それらの値は等しくないとみなします。一方、値の型は同じだが数値ではない場合は、同じ値であれば等しいとみなします。また、どちらの値も数値である場合は、どちらも NaN ではなく同じ値である、あるいは一方が +0 かつもう一方が -0 であるときは等しいとみなします。

    var num = 0;
var obj = new String("0");
var str = "0";
var b = false;

print(num === num); // true
print(obj === obj); // true
print(str === str); // true

print(num === obj); // false
print(num === str); // false
print(obj === str); // false
print(null === undefined); // false
print(obj === null); // false
print(obj === undefined); // false

    厳密な等価性は、たいてい正しい等価演算を行うものです。数値以外のあらゆる値において、これは「値が自分自身と等しい」という明快な方式を用います。数値においては、2 つの特殊なケースが取り込まれた若干異なった方式を使用します。ひとつは、浮動小数点数の 0 は正負両方が存在することです。これはある種の数学的な解答を表すために役立ちますが、ほとんどの場合は +0 と -0 の違いを意識しないため、厳格な等価性ではこれらを同じ値として扱います。もうひとつは、浮動小数点数には非数 NaN の概念があることです。これは数学的に明確ではない問題、例えば正の無限大に負の無限大を加算するなどの回答を表すものです。厳格な等価性では NaN を他のどの値 (自分自身も含む) とも等しくないものとして扱います。(x が NaN である場合に、(x !== x) が true になるだけです)

    == による寛容な等価性

    寛容な等価性は、双方の値を一般的な型に変換した後で 2 つの値が等しいかを比較します。(一方あるいは双方が変換される) 変換処理後に、=== と同じ方法で等価性を比較します。寛容な等価性は対称的であり、任意の値 A および B について、A == B と B == A の意味は常に同一です。

    さまざまな型に対して、比較演算は以下のように行います:

      オペランド B
        Undefined Null Number String Boolean Object
    オペランド A Undefined true true false false false IsFalsy(B)
    Null true true false false false IsFalsy(B)
    Number false false A === B A === ToNumber(B) ToNumber(B) === A ToPrimitive(B) == A
    String false false B === ToNumber(A) A === B ToNumber(A) === ToNumber(B) ToPrimitive(B) == A
    Boolean false false ToNumber(A) === B ToNumber(A) === ToNumber(B) A === B false
    Object IsFalsy(A) IsFalsy(A) ToPrimitive(A) == B ToPrimitive(A) == B false

    A === B

    上の表で ToNumber(A) は、比較前に引数を数値に変換しようとします。これは +A (単項演算子 +) と同じ動作です。ToPrimitive(A) は、オブジェクト A の A.toString メソッドおよび A.valueOf メソッドによる変換処理を行って、オブジェクトからプリミティブ値への変換を試みます。

    伝統的におよび ECMAScript によると、すべてのオブジェクトは undefined および null と寛容に等しくないとされています。しかしほとんどのブラウザは特定の状況において値 undefined をエミュレートするために、ごく一部のオブジェクト (特にあらゆるページの document.all オブジェクト) に対してはそれを認めています。寛容な等価性がそれに該当します。従って IsFalsy(A) メソッドで true と評価される場合に限り、A は undefined をエミュレートするオブジェクトです。それ以外のオブジェクトはすべて、undefined および null と寛容な等価性がありません。

    var num = 0;
var obj = new String("0");
var str = "0";
var b = false;

print(num == num); // true
print(obj == obj); // true
print(str == str); // true

print(num == obj); // true
print(num == str); // true
print(obj == str); // true
print(null == undefined); // true

// both false, except in rare cases
print(obj == null);
print(obj == undefined);

    寛容な等価性を使用することはよい考えでないと考える開発者もいます。厳格な等価性による比較は結果が容易に予測でき、また型変換がないため評価が早く行われます。

    同一値の等価性

    同一値の等価性 (same-value equality) は、最終的に適用されます。これはすべての状況において、2 つの値が機能的に同一かを判断します。(これはリスコフの置換原則を実証します) イミュータブルなプロパティを変化させようとするときが実例のひとつです:

    // Number コンストラクタに immutable な NEGATIVE_ZERO プロパティを追加
Object.defineProperty(Number, "NEGATIVE_ZERO",
                      { value: -0, writable: false, configurable: false, enumerable: false });

function attemptMutation(v)
{
  Object.defineProperty(Number, "NEGATIVE_ZERO", { value: v });
}

    イミュータブルなプロパティを実際に変更される場合は Object.defineProperty が例外を発生させますが、実際の変化が要求されていない場合は何も発生しません。v が -0 であれば変化は求められず、またエラーが発生しません。しかし v が +0 であれば、Number.NEGATIVE_ZERO はイミュータブルな値を持つものではなくなります。内部ではイミュータブルなプロパティが再定義されると、新たに指定された値と現在の値が同一値の等価性によって比較されます。

    同一値の等価性は Object.is メソッドで提供します。
