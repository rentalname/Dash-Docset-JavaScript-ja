    Storage Inspector

    サイドバーを表示

    Storage Inspector は Firefox 34 の新機能です。

    Storage Inspector はデフォルトで無効です。開発ツールのオプションで有効化できます。

    Storage Inspector で、Web ページが使用できるさまざまな種類のストレージを調査できます。現在は、以下の種類のストレージの調査に使用できます:

    • Cookie - ページ自身およびページ内の iframe が作成したすべての Cookie を確認できます。ネットワーク呼び出しの応答の一部として作成された Cookie も表示しますが、ツールを開いている間に発生した呼び出しによるものだけが対象です。
    • Local Storage - ページ自身およびページ内の iframe が作成したすべての Local Storage アイテムを確認できます。
    • Session Storage - ページ自身およびページ内の iframe が作成したすべての Session Storage アイテムを確認できます。
    • IndexedDB - ページ自身およびページ内の iframe が作成したすべての IndexedDB データベースで、データベースのオブジェクトストアおよびオブジェクトストア内に保存しているアイテムを確認できます。

    Storage Inspector は当分の間、読み取り専用のビューのみでストレージを表示します。将来のリリースでストレージコンテンツの編集を可能にするよう、作業しています。

    Storage Inspector を開く

    Storage Inspector を有効化すると、Firefox メニューパネル (メニューバーを表示している場合や Mac OS X では ツール メニュー) 内の Web 開発サブメニューで "Storage Inspector" を選択して開くことができます。または、キーボードショートカット Shift + F9 も使用できます。

    Storage Inspector がアクティブな状態で、ツールボックスがブラウザウィンドウの下部に現れます。開発ツールボックスでは "Storage" という名称です。

    Storage Inspector のユーザインターフェイス

    Storage Inspector の UI は 3 つの主要コンポーネントに分かれます:

    • ストレージツリー
    • テーブルウィジェット
    • サイドバー

    ストレージツリー

    ストレージツリーは、Storage Inspector が調査可能なすべてのストレージタイプを一覧表示します。各タイプについて、使用可能な生成元を一覧表示します。生成元は、ストレージタイプによりドメインまたはホスト名になります。Cookie では、プロトコルが生成元を区別しません。IndexedDB や local storage では、生成元がプロトコルとホスト名の組み合わせになります。例えば "http://mozilla.org" と "https://mozilla.org" は異なる 2 つの生成元になりますので、local storage のアイテムは双方で分け合うことができません。

    Cookie、local storage、session storage のアイテムは生成元に直接関連づけられますが、IndexedDB はもう一つ階層があります。すべての IndexedDB アイテムはデータベースに関連づけられたオブジェクトストアに割り当てられており、最終的にデータベースが生成元に紐付けられます。

    ツリー内の各項目をクリックすると、子項目を展開または折りたたみできます。ツリーは生きていますので、新たな生成元が追加される (例えば iframe を追加するなど) と、自動的に各ストレージタイプに追加されます。ツリー項目をクリックすると、右側にあるテーブルのデータが変わります。例えばストレージタイプ Cookie 配下の生成元をクリックすると、ツリー内でそのドメインに属するすべての Cookie を表示します。あるいはオブジェクトストアの項目をクリックすると、そのオブジェクトストア内にあるすべての IdexedDB アイテムを表示します。同様にデータベースをクリックすると、そのデータベース内の全 object store を表示します。

    テーブルウィジェット

    テーブルウィジェットは、選択したツリー項目 (生成元やデータベース) に関するすべてのアイテムを表示する場所です。ストレージタイプやツリー項目に応じて、テーブルの列の数が変わります。

    テーブルウィジェットのすべての列はリサイズ可能であり、またヘッダのコンテキストメニューから、2 列だけ見えている状態まで列を隠すことができます。

    Cookie

    ストレージツリーで Cookie ストレージ内の生成元を選択すると、テーブルにその生成元向けの Cookie を表示します。テーブルには以下の列があります:

    • Name - Cookie の名称
    • Path - Cookie の path プロパティ
    • Domain - Cookie のドメイン
    • Expires On - Cookie の有効期限。セッション Cookie である場合は、この値は "Session" になります。
    • Created On - Cookie が作成された日時
    • Last Accessed On - Cookie が最後に読み取られた日時
    • Value - Cookie の値
    • isDomain - ドメイン Cookie であるか。ドメイン Cookie である場合は、domain の値が "." から始まります。
    • isSecure - セキュア Cookie であるか
    • isHttpOnly - HTTPOnly の Cookie であるか

    Local storage / Session storage

    Local Storage および Session Storage の生成元を選択すると、Local Storage および Session Storage に関するすべてのアイテムの名称と値をテーブルに表示します。

    IndexedDB の生成元

    ストレージツリーで Indexed DB ストレージ内の生成元を選択すると、その生成元向けのすべてのデータベースの詳細情報をテーブルに表示します。データベースの詳細情報は以下のとおりです:

    • Database Name - データベースの名称
    • Origin - データベースの生成元
    • Version - データベースのバージョン
    • Object Stores - データベース内にあるオブジェクトストアの数

    IndexedDB データベース

    ストレージツリーで IndexedDB のデータベースを選択すると、保存されているすべてのオブジェクトストアの詳細情報をテーブルに表示します。オブジェクトストアの詳細情報は以下のとおりです:

    • Object Store Name - オブジェクトストアの名称
    • Key - オブジェクトストアの keyPath プロパティ
    • Auto Increment - オートインクリメントが有効であるか
    • Indexes - オブジェクトストアのインデックスの配列を下図のように表示する

    IndexedDB オブジェクトストア

    ストレージツリーでオブジェクトストアを選択すると、オブジェクトストア内のすべてのアイテムをテーブルに表示します。すべてのアイテムは、キーとキーに関連づけられた値があります。

    サイドバー

    テーブルウィジェットで任意の行を選択すると、その行の詳細情報を表示するサイドバーが開きます。Cookie を選択すると、その Cookie に関するすべての詳細情報を表示します。

    サイドバーでは Cookie、Local Storage、IndexedDB のアイテムの値を解析して、単なる文字列を有意義なものに変換します。例えば '{"foo": "bar"}' のように stringify 処理が施された JSON を元の JSON である {foo: "bar"} として表示したり、1~2~3~4 のようにキーで区切られた値を配列 [1, 2, 3, 4] として表示したりします。さまざまな値を解釈したときのスクリーンショットを以下に示します:

    stringify 処理を施された JSON を元の JSON として、サイドバーの Parsed Value セクションに表示する

    キーと値のペアを持つ文字列を JSON として、サイドバーの Parsed Value セクションに表示する

    キーで区切られた値を持つ文字列を配列として、サイドバーの Parsed Value セクションに表示する
