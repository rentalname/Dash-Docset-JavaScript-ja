    Performance

    サイドバーを表示

    このツールは Firefox 34 の新機能です。

    パフォーマンスツールは、JavaScript サンプリングプロファイラを置き換えるツールです。このツールは最新のサンプリングプロファイラだけでなく、応答性を測定するためのフレームレートタイムラインも搭載しました。将来のリリースではさらに機能を追加する予定です。

    パフォーマンスツールを開く

    "Web 開発" メニューから "Performance" を選択すると、プロファイラを起動できます。"Web 開発" メニューは、Linux や OS X では "ツール" メニューの配下に、Windows では "Firefox" メニューの直下にあります。または、Shift + F5 でもツールを開くことができます。

    プロファイルの作成

    ストップウォッチのアイコンをクリックすると、新たなプロファイルの記録を始めます。もう一度クリックすると、記録を終了します。プロファイルは直ちに開きます。プロファイルを JSON ファイルで保存する、あるいは保存済みのプロファイルをインポートすることができます。左側のサイドバーで、開いている複数のプロファイルを切り替えることができます。

    プロファイルの分析

    プロファイルは以下のようなものです:

    水平に積み重なった 4 つの主要なセクションで構成されます:

    1. 拡大・縮小に使用できるパンくずリストのセット

    2. フレームレートタイムライン

    3. JavaScript サンプリングプロファイラのタイムライン

    4. JavaScript サンプリングプロファイラの詳細な出力

    フレームレートタイムライン

    フレームレートタイムラインは、プロファイラに含まれる時間帯でのブラウザの応答性の概要を示します。

    フレームは、ブラウザが何らかのイベントへの応答として自身を更新するために必要な作業が要約されます。例えば、ページ上のある要素にマウスポインタを載せると要素の外見を変更する JavaScript が動作してリフローや再描画が発生するとすれば、それらすべての作業がフレーム内で完了しなければなりません。ブラウザがフレームを処理する時間よりそれらの作業にかかる時間が長い場合は、ブラウザが反応しないように見えるでしょう (janky)。

    パフォーマンスツールはブラウザがフレームを処理したときのタイムスタンプを取得して、フレームレートを追跡するために使用します:

    横軸はプロファイルを採取した期間の経過時間です。また、目標フレームレート (常に毎秒 60 フレーム)、平均フレームレート、最小フレームレートを注釈として表示します。

    サンプリングプロファイラ

    JavaScript プロファイラは JavaScript エンジンの状態を定期的にサンプリングして、その時点のコード実行のスタックを記録します。統計的に、個々の関数を実行しているときに取得したサンプル数はブラウザが実行にかけた時間に対応しますので、コード内のボトルネックを発見できます。

    例えば、以下のようなプログラムについて考えてみましょう:

    function doSomething() {
  var x = getTheValue();
  x = x + 1;                // -> A (トップレベルから呼び出し)
  logTheValue(x);
}

function getTheValue() {
  return 5;                 // -> B (doSomething() から呼び出し)
}

function logTheValue(x) {
                            // -> C (doSomething() から呼び出し)
                            // -> D (doSomething() から呼び出し)
                            // -> E (トップレベルから呼び出し)
 console.log(x);
}

doSomething();

logTheValue(6);

    プロファイラをアクティブにしてこのプログラムを実行したら、実行時にプロファイラは上記のインラインコメントで示したように 5 つのサンプルを取得します。

    4 つは doSomething() の内部で取得します。そのうち A は、doSomething() の直下です。B は getTheValue() の内部、C と D は logTheValue() の内部になります。最後にプログラムはトップレベルで logTheValue() を呼び出しており、ここでもサンプルを取得しています。従って、プロファイルは以下のように 5 つのスタックトレースで構成されます:

    サンプル A: doSomething()
サンプル B: doSomething() > getTheValue()
サンプル C: doSomething() > logTheValue()
サンプル D: doSomething() > logTheValue()
サンプル E: logTheValue()

    これは私たちに何も伝えられない不十分なデータであることが明らかですが、より多くのサンプルにより、logTheValue() がコード内のボトルネックであると断定できるかもしれません。

    パフォーマンスツールはこれらのデータを、タイムラインと詳細ビューの 2 通りの方法で視覚化します。

    サンプリングプロファイラタイムライン

    サンプリングプロファイラタイムラインは、プロファイラに含まれる時間帯での JavaScript の実行状況の概要を示します。

    フレームレートタイムラインと同様に、横軸はプロファイルをを採取した期間の経過時間です。またサンプルは縦棒グラフとして、取得した順に右から左へ並べて表示します:

     

    それぞれの棒の高さは、その時点のコールスタックの深さを表します。

    パフォーマンスツールはコールスタックを調査して、JavaScript コードが行っていた処理の種類を明らかにします。そして、処理の種類に応じて棒の色を分けます:

    Network ネットワーク要求や応答の送信と処理
    JIT JavaScript コードのコンパイル
    GC ガベージコレクション
    Inputs & events マウスイベントや DOM イベントなどのイベント
    Styles CSS の処理
    Graphics WebGL だけでなく、リフローや再描画の処理も含む
    Storage もっとも多いものは IndexedDB
    Gecko 他のカテゴリにあてはまらないすべての処理

    凡例にあるカテゴリの色見本にマウスポインタを載せると、他のカテゴリを淡色で表示します:

    このビューはフレームレートタイムラインと並んでいますので、フレームレートの遅さと特定の JavaScript 処理とを関係づけることができるでしょう。

    サンプリングプロファイラの詳細情報

    サンプリングプロファイラの詳細情報ペインは以下のようなものです:

    これは、収集したサンプルを表形式で表示します。"Calls" 列は、"Function" 列の関数をコールスタック内に存在しているサンプルの数です。

    例えば前出のプロファイリングの例では、"Calls" 列と "Functions" 列の対応は以下のようになります:

    4  doSomething()
2  > logTheValue()
1  > getTheValue()
1  logTheValue()

    logTheValue() は 2 か所から呼び出されていますので Call が 2 になることに注意してください。

    "Time" 列は取得したサンプルの数によって統計的に推定した、関数内で費やした時間の合計値です。この値は、サンプルの数と完全には相関していません。これはサンプルを常に正確な時刻で取得しているわけではないことから、不規則さを補正しようとしているためです。

    "Cost" 列は、"Time" 列をパーセント値に変換した値です。

    "Function" 列は関数名、ソースファイル名、行番号、ファイルを提供したドメインの情報を含みます。ファイル名をクリックすると、そのファイルをデバッガで開きます。

    行をクリックすると、その関数が現れたすべてのサンプルをタイムラインビューでハイライト表示します:

    行にマウスポインタを載せると、右端に拡大鏡のアイコンが表示されます。拡大鏡をクリックすると、その行がルートになります。すなわち、その行と配下の行だけが存在するようなプロファイルができます:

    これは、プロファイル内で対象のサンプルが存在する部分を拡大していることにもなります。

    拡大

    フレームレートタイムラインまたはサンプリングプロファイラタイムラインでクリックアンドドラッグを行うと、以下のようになります:

    • ドラッグした範囲のタイムラインをハイライト表示します。
    • サンプリングプロファイラの詳細ビューでは、ドラッグした範囲内で記録されたサンプルのみを表示します。
    • 上部のツールバーに "+" マークが現れます:

    "+" をクリックすると、ハイライト表示している範囲をタイムライン全体に表示します。上部のツールバーには、その範囲を別のパンくずリストとして表示します:

    パンくずリストを使用して、プロファイル全体と選択範囲とを切り替えできます。選択範囲の一部をさらに拡大することもでき、それはツールバーで第 3 のパンくずリストとして表示します:
