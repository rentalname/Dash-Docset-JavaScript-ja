    デバッガの起動

    Web コンテンツのデバッグ

    Firefox で直接実行している Web コンテンツ (ローカルのコンピュータに保存されているかサーバ上にあるかを問わず) をデバッグするには、Web 開発メニューで「デバッガ」を選択します。

    要 Firefox 19.0

    リモートデバッグ

    Firefox および Firefox OS でデバッグをサポートするように設定すれば、Firefox OS デバイスや Firefox OS Simulator で実行しているコードをデバッグするための、リモートデバッグが使用可能になります。

    Forefox OS や Android デバイスで実行している JavaScript をデバッグするためには、デバイスが USB で接続されていて、lsusb および adb を参照できることが必要です。そして、ターミナルウィンドウを開いて以下のコマンドを実行することにより、デバイスにとって適切な TCP ポートへフォワードすることが必要です:

    adb forward tcp:6000 tcp:6000

    注意: デバイスを再起動したときは、再度ポートをフォワードしなければなりません。

    Web 開発メニューでリモートデバッガを選択すると、リモートデバッガを開くことができます。リモートデバッガは、デバッグのために接続したい IP アドレスとポートを指定するよう求めます。デバイス側でこれらの設定を変更していない限り、localhost:6000 のままにしておきます。OK をクリックして少し待つと、デバッガが起動します。

    この時点で、デバッガはまだ何も行いません。これは、あなたがブレークポイントなどを何も設定していないためです。デバイスは鼻歌を歌い続けているような状態です。以下の章で、実際のデバッガ使用について扱います。

    デバッガツールバー

    ツールバーには、いくつかのコントロールがあります:

    停止/復帰 (F6)
    デバッグしているスクリプトの実行を停止および再開します。
    ステップオーバー (F7)
    JavaScript コードで現在の行をステップ実行します。
    ステップイン (F8)
    JavaScript で現在の行にある関数呼び出しにステップインします。
    ステップアウト (Shift-F8)
    現在の関数を抜けるまでスクリプトを実行します。
    スクリプト選択 (Ctrl-P または Cmd-P)
    このポップアップメニューで、コードを読んだりブレークポイントを設定したりなどするために、実行しているスクリプトからひとつを選択できます。
    スクリプトフィルタ
    スクリプトフィルタ (検索ボックスを兼ねる) は文字列を入力することで、スクリプト選択に表示したいスクリプトをフィルタできます。Using the script filter をご覧ください。
    ペインの拡張
    サイドバーパネルを表示したり隠したりするため、ウィンドウを拡張あるいは縮小します。
    デバッガオプション
    デバッガの設定が可能なメニューがポップアップします。Debugger options をご覧ください。

    注記: Web コンテンツのデバッガではツールバーの左端に閉じるボタンがあります。こちらでデバッガを閉じることができます。

    スクリプトフィルタの使用

    ツールバー内のスクリプトフィルタをクリックして、マッチするスクリプトだけをスクリプト選択に表示するように、フィルタするための文字列を入力できます。またご覧のとおり、このボックスをクリックすると、検索ボックスとして使用したり他の機能を使用したりできるようにする特殊演算子が表示されます。各々の特殊演算子には、自動的にフィルタボックスへ移動して演算子を入力するキーボードショートカットがありますので、あとは入力を始めるだけです。

    すべてのファイルを検索 - !(Cmd-Opt-F)
    実行しているスクリプト全体から、入力したテキストを検索します。検索するとツールバーの直下にマッチしたファイルをすべて表示するペインを追加します。各ファイルの項目では、ファイル内でマッチした箇所を閲覧できるボックスが表示されます。
    このファイルを検索 - # (Cmd-F)
    現在表示しているファイル内で、入力したテキストを検索します。
    指定行へ移動 - : (Cmd-J)
    ボックスへ入力した値の行へ移動します。
    変数を検索 - * (Cmd-Opt-V)
    指定した文字列にマッチする変数だけが含まれるよう、変数一覧をフィルタします。

    デバッガのオプション

    Debugger Options のアイコンをクリックすると、デバッガの動作を調整できる項目をもつポップアップメニューが表示されます。

    例外発生で停止
    このオプションが有効であるときは、JavaScript の例外が発生した際にスクリプトの実行が自動的に停止します。
    起動時にデバッガを開く
    このオプションが有効であるときは、デバッガを起動した初期状態で 2 つのサイドバーペインが表示されている状態になります。デフォルトでは表示されません。
    Show hidden properties
    このオプションを有効にしない限り、隠し JavaScript プロパティ (すなわち、列挙可能ではないもの) は表示されません。
    変数フィルタボックスを表示
    このオプションを有効にすると変数ペインに "変数を検索" 検索ボックスが追加されますので、表示されている変数一覧のフィルタが可能です。

    デバッガの使用

    JavaScript デバッガは、現代のデバッガに期待される標準機能のほとんどを備えています。

    ブレークポイント

    ブレークポイントを設定したいファイルをスクリプト選択のドロップダウンメニューから選択して (そのファイルを表示していない場合)、ブレークポイントを設定したい行で行番号列にある行番号そのものの左側をクリックすることで、ブレークポイントを設定できます。また、ブレークポイントを作成したい行のコード部分で右クリックすると、ブレークポイント (Ctrl+B または Cmd-B) や条件付きブレークポイント (Ctrl+Shift+B または Cmd-Shift-B) を作成できるコンテキストメニューが表示されます。

    一般的なデバッガのように、ステップ実行するためにツールバーの復帰、ステップオーバー、ステップイン、ステップアウトボタンを使用できます。次に実行されるコードの行の左側に、緑色の矢印が表示されます。もちろん、ブレークポイントの設定や削除、変数の調査などを行うこともできます。

    ソースコードの右側にある列には、クリックすることで対応するブレークポイントへ素早くスクロールできる、青色の印が表示されます。

    条件付きブレークポイント

    条件付きブレークポイントは、与えられた JavaScript 式が真であるときにのみ発生する特別なブレークポイントです。条件付きブレークポイントを設定するには、コードの行を右クリックして "条件付きブレークポイントを設置" を選択します (または Ctrl+Shift+B/Cmd-Shift-B を押下します)。通常のブレークポイントを作成してから、デバッガウィンドウの左下にあるブレークポイントの一覧で右クリックすることで条件を追加することもできます。

    ブレークポイントの管理

    デバッガウィンドウの左下にあるブレークポイントの一覧を使用して、ブレークポイントを管理することができます。ブレークポイントにあるチェックボックスの On/Off を切り替えると、ブレークポイントの有効・無効が切り替わります。条件付きブレークポイントをクリックすると Conditional breakpoints で見たように、条件付きブレークポイントを編集するパネルがポップアップします。

    ブレークポイントを右クリックすると、以下のオプションを設定できます:

    すべてのブレークポイントを取り除く
    ブレークポイントをすべて削除します。
    すべてのブレークポイントを有効にする
    すべてのブレークポイントを有効にします。これは、すべてのブレークポイントにあるチェックボックスをオンにすることのショートカットです。
    すべてのブレークポイントを無効にする
    すべてのブレークポイントを (削除せずに) 無効にします。これは、すべてのブレークポイントをにあるチェックボックスをオフにすることのショートカットです。
    ほかのブレークポイントを有効にする
    右クリックしたもの以外の、すべてのブレークポイントを有効にします。
    ほかのブレークポイントを無効にする
    右クリックしたもの以外の、すべてのブレークポイントを無効にします。
    ほかのブレークポイントを取り除く
    右クリックしたもの以外の、すべてのブレークポイントを削除します。
    条件付きブレークポイントを設定する
    右クリックした条件付きブレークポイントを設定します。ブレークポイントが条件付きではない場合は、条件を追加して条件付きブレークポイントに変更できます。
    ブレークポイントを無効にする
    右クリックしたブレークポイントを無効にします。
    ブレークポイントを有効にする
    右クリックしたブレークポイントを有効にします。

    変数パネル

    デバッガの右側パネルの大部分は、現在見ているスコープで使用できる変数で占められています。ご覧いただいているように、ここには現在実行している関数 (この図では qs_handleEvent()) および呼び出し元関数 (この図では ut_show()) のローカルスコープとグローバルスコープ (この図では Window オブジェクト) の変数が表示されます。

    各々のオブジェクトは三角印をクリックしてメンバを表示することができ、これによってオブジェクトの内容を明らかにします。変数の現在の値をクリックして新しい値を入力することで、変数の値を変更できます。例えば geolocationEnabled の隣にある "true" をクリックすると、値を false に変更するために "false" を入力するといったことが可能です。値を変更された変数は、以下のように黄色で強調されます:

    変数名にマウスポインタを載せると、その変数に関する付加情報を提供するツールチップが表示されます。例えば evt オブジェクトをポイントすると "configurable enumerable writable" と表示されます。これは evt オブジェクトが configurable ではありませんが、 enumerable かつwritable であることを示します。これらのプロパティの意味について、詳しくは Object.defineProperty() をご覧ください。

    スクリーンショットにあるとおり filter variables box を有効にすると、実際に確認したいものだけを表示するように大量のリストを限定して小さくすることができます。この検索は再帰的ですので、サブオブジェクトにも渡って検索されます。例えば "blue" と入力すると、一覧が qs_handleEvent() 関数のスコープにある this.bluetooth オブジェクトとグローバルスコープの Bluetooth および BluetoothTransfer オブジェクトに限定されます。

    Watch expressions

    Watch expressions は、実行が停止するたびに評価される式です。よって、これらの式の結果を調査できます。これらは、コード内にあるが必ずしも調査する準備ができていない不変量の調査を可能にする点で有用です。watch expression を追加するには "Add watch expression" と表示されているボックスをクリックして、コードのステップ実行に応じてモニタリングする出力内容を表す JavaScript 式を入力します。

    そして、コードの実行を開始します。コードのステップ実行を始めるまで watch expression は何も行いませんので、ブレークポイントに達するまでは何も起きません。ブレークポイントに達すると、アクティブな式とその値を表示するボックスが現れます。

    例えばあるコードをステップ実行して、変数 a を 2 倍した値をつまらない "思考" なしにすばやく知りたいとします。"Add a watch expression box" に式 a*2 を入力して Enter キーを押下して、適切なブレークポイントを設定して、コードを実行します。

    コードをステップ実行して、値の変化に応じた式の結果をウォッチできます。値が変わるたびに、ボックスが一時的に黄色に光ります。式の隣にある "x" 印のアイコンをクリックして watch expression を削除でき、またもちろん、watch expression は同時に複数設定できます。

    デバッガの停止

    デバッグが完了したときは好みに応じて、Setting アプリを起動して Device Information を選択することで、デバイスでのリモートデバッグを無効にできます。このページに、More Information ボタンがあります。これをタップして、移動先のページで下にスクロールして "Developer" をタップします。ここには開発オプションの一覧があります。ここで Remote Debugging をオフにします。無効化を望まないのであれば、この作業を行ってはいけません。

    注記: 2012 年 11 月 29 日の Nightly ビルドから、デバイスのリモートデバッグサポートのオン・オフ切り替えでは再起動が不要になっています。
