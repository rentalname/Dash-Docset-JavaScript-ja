    WebIDE は Firefox 34 以降で使用できます。

    Firefox 33 でも設定を変更することで使用できます。about:config を開いて devtools.webide.enabled という設定項目を探し、true に設定してください。

    WebIDE はアプリマネージャの後継品です。WebIDE はアプリマネージャと同様に、Firefox OS Simulator や Firefox OS の実機を使用して Firefox OS アプリの実行やデバッグが可能です。

    しかしそれだけでなく、Firefox OS アプリの作成や開発のための編集環境も提供します。これにはアプリ内の全ファイルのツリービューや編集および保存機能、そしてアプリを作り始めるのに役立つ 2 つのアプリテンプレートもあります。

    WebIDE では、始めに 1 つ以上の実行環境のセットアップを行います。実行環境は、アプリの実行やデバッグを行う環境です。実行環境は USB でパソコンに接続した Firefox OS デバイスか、パソコン自体にインストールした Firefox OS Simulator を使用できます。

    次に、アプリを作成するか既存のアプリを開きます。アプリを新規作成する場合は、アプリを作り始めるのに必要なディレクトリ構造や最小限の標準モデルを収めたテンプレートか、Privileged API を使用する方法を示すより充実したテンプレートを使用します。WebIDE はアプリ内のファイルをツリーに表示します。また内蔵のソースエディタでそれらの編集や保存が可能です。もちろん、内蔵のエディタを使用しないことも可能です。アプリの開発は WebIDE の外部で行い、デバッグにのみ WebIDE を使用できます。

    そして、アプリをいずれかの実行環境にインストールして実行できます。また実行中のアプリの調査や変更を行うために、いつもの開発ツール一式であるインスペクタ、コンソール、JavaScript デバッガなどを開くことができます。

    システム要件

    WebIDE を使用してアプリの開発やデバッグを行うには、Firefox 33 以降が必要です。Firefox OS デバイス実機でテストを行うには、Firefox OS 1.2 以降が動作するデバイスと USB ケーブルが必要です。

    Firefox OS 1.2 以降を対象にする場合のみ、WebIDE を使用できます。

    WebIDE を開く

    Web 開発メニューで "WebIDE" をクリックすると、WebIDE が開きます。また、キーバインド Shift-F8 も使用できます。左側のドロップダウンメニュー "Open App" で、既存のアプリの起動や新しいアプリの作成ができます。右側のドロップダウンメニュー "Select Runtime" で、実行環境の選択や新しい実行環境のセットアップができます。

    中央にあるボタンでアプリの実行・停止・デバッグを行います。これらのボタンはアプリを開いていて、かつ実行環境が選択済みである場合にのみ使用できます。

    実行環境のセットアップ

    "Select Runtime" ドロップダウンメニューでは、実行環境を 3 種類にグループ分けしています:

    • USB devices: USB で接続した Firefox OS デバイス
    • Simulators: インストール済みの Firefox OS Simulator のインスタンス
    • Custom: 任意のホスト名とポートを使用して WebIDE に接続する

    初めてドロップダウンメニューをクリックしたときは、実行環境がない状態でしょう:

    ここでは、実況環境を追加する方法を説明します。

    Firefox OS デバイスの接続

    Firefox OS デバイスを接続できるようにするために、以下のセットアップ作業が必要です:

    • Firefox OS のバージョンを確認する: デバイスで Firefox OS 1.2/Boot2Gecko 1.2 以降を実行しているかを確認してください。バージョンはデバイスで Settings アプリを起動して、Device Information > Software を選択すると確認できます。バージョンが満たない場合は開発端末ガイドでお持ちのデバイスを探して、アップグレード手順に従ってください。
    • リモートでバッグの有効化: デバイスで Settings アプリを起動して、Device information > More information > Developer と進んでください。
      • Firefox OS 1.3 以前: "Remote Debugging" というチェックボックスがあります。Check it.
      • Firefox OS 1.4 以降: "Remote Debugging" で ADB のみ有効化するか、ADB と開発ツールを有効化するかが問われます。"ADB and DevTools" を選択してください。
    • デバイスのスクリーンロックの無効化: デバイスの Settings アプリで、Screen Lock の Lock Screen チェックボックスのチェックを外してください。スクリーンがロックされると電話機との接続が失われるため、デバッグが行えない状態になります。
    • プリインストールアプリを含む、公認アプリのデバッグを行う場合: 公認アプリのデバッグをご覧ください。

    Linux の注意事項:

    • Android デバイスのセットアップガイドのステップ 3 に記載されているとおりに、udev のルールファイルを追加してください。idVendor 属性の値は Geeksphone であれば "05c6"、他のベンダーの値はこちらのページに記載しています。

    Windows の注意事項:

    • Android デバイスのセットアップガイドのステップ 3 に記載されているとおりに、ドライバのインストールが必要です。Geeksphone のデバイスのドライバは、Geeksphone の Web サイトで入手できます。Windows 8 はデフォルトで、未署名のドライバをインストールできません。"How to install an unsigned driver on Windows 8" のチュートリアルをご覧ください。
    • すべての手順を行っても WebIDE がデバイスを認識しない場合は、adb_usb.ini の編集が必要である可能性があります。

    他の Android デバイスを接続している場合は、取り外してください。そして、USB でデバイスをコンピュータと接続してください。"USB DEVICES" の配下に、デバイスが表示されるでしょう:

    デバイスが表示されない場合は、トラブルシューティングのページをご覧ください。

    Simulator の追加

    Firefox OS Simulator は Firefox OS の上位層であり、パソコン上で動作して Firefox OS デバイスのシミュレーションを行います。これは Firefox OS デバイスと同じサイズのウィンドウで動作し、Firefox OS のユーザインターフェイスや同梱アプリを含み、そして Firefox OS デバイスの API の多くをシミュレーションします。

    これは多くの場合、アプリのテストやデバッグに実機が必要ないということです。

    Simulator はサイズが大きいため Firefox に同梱せず、Firefox のアドオンとして提供しています。実行環境のドロップダウンメニューで "Install Simulator" をクリックすると、Firefox OS のさまざまなバージョンの Simulator をインストールできるページが開きます。

    Simulator はいくつでもインストールできます。ただし、Simulator はサイズが大きいためダウンロードに時間がかかりますのでお待ちください。Simulator をインストールしたら "Extra Components" を閉じてください。Runtimes のドロップダウンメニューの項目として、インストールした Simulator が表示されます:

    Simulator について詳しくは、ドキュメントをご覧ください。

    カスタム実行環境

    カスタム実行環境により、任意のホスト名とポート番号を使用してリモートのデバイスに接続できます。

    内部で Firefox OS デバイスや Android デバイスは、パソコンと Android Debug Bridge あるいは ADB と呼ばれるプログラムを用いて接続しています。デフォルトで WebIDE は、ADB Helper を使用します。これは ADB のインストールやポートフォワーディングの設定により手続きを単純化しますので、Firefox のデスクトップツールとデバイスがメッセージを交換できるようになります。

    これはほとんどの場合に便利ですが、ADB を WebIDE の外部で使用したい場合があるかもしれません。例えば、コマンドラインから直接 ADB を実行することがあるでしょう。その場合は adb forward コマンドを使用して、ホスト名とポート番号を指定することによりデバイスと接続します。

    さらに WebIDE も接続したい場合は ADB Helper アドオンを無効化したうえでカスタム実行環境により、adb forward に渡すホスト名とポート番号を入力して WebIDE を接続します。

    また、ADB Helper は Android 版 Firefox への接続が未サポートですので、Android 版 Firefox に WebIDE を接続したい場合はポートフォワーディングを設定したうえで、カスタム実行環境を使用して接続することが必要です。詳しくは ADB を使用して Android 版 Firefox に接続する方法のドキュメントをご覧ください。

    実行環境の設定

    実行環境をセットアップすると、"Select Runtime" メニューで選択できるようになります。

    • Simulator を選択すると、WebIDE は Simulator を起動します。
    • Firefox OS デバイスを選択すると、WebIDE はデバイスに接続します。デバイスでは接続を望むかを問い合わせるダイアログが表示されます。"OK" を押してください。

    以上で、WebIDE のツールバーの中央にある "play" ボタンが使用可能になります。このボタンをクリックして、選択した実行環境でアプリのインストールや実行を行います。

    実行環境のアクション

    実行環境を選択すると、Runtimes ドロップダウンメニューに項目が 3 個増えます:

    • Runtime Info: 選択している実行環境の情報
    • Permissions Table: 選択中の実行環境におけるアプリの許可設定を要約した表。API ごと、およびアプリの種類ごとに、アクセスが可能 (✓)、不可 (✗)、ユーザに問い合わせるか (!) を示します。
    • Screenshot: 実行環境のスクリーンショットを取得するコマンド

    アプリの作成と起動

    "Open App" メニューの配下に 3 個の項目があります: アプリの新規作成、パッケージ型アプリを開く、ホスト型アプリを開く。

    アプリの新規作成

    "New App..." を選択して、アプリを新規作成します。"Privileged Empty App" と "Privileged App" の 2 つのテンプレートから選択できるダイアログが現れます。

    どちらのテンプレートも Mozilla のアプリテンプレートコレクションから提供しており、アプリの作成に必要な基本構造を提供します。"Privileged App" は、アプリがクロスオリジンのコンテンツを読み込むために許可設定をどのようにすればよいかを示します。

    テンプレートを選択すると、アプリの名称およびファイルを保存するディレクトリを問われます。そして、新規アプリをプロジェクトエディタで開きます。

    パッケージ型アプリを開く

    "Open Packaged App..." を選択して、パッケージ型アプリを開きます。こちらを選択すると、アプリのマニフェストがあるディレクトリを求められます。そして、アプリがプロジェクトエディタで開かれます。

    ホスト型アプリを開く

    "Open Hosted App..." を選択して、ホスト型アプリを開きます。こちらを選択すると、アプリのマニフェストを示す URL を求められます。そして、アプリがプロジェクトエディタで開かれます。

    アプリの編集

    プロジェクトエディタは、アプリを編集するための環境です。左側にアプリ内の全ファイルを表示するツリービューがあり、コンテキストメニューを使用してファイルの追加や削除が可能です。また、右側にエディタのペインがあります。

    アプリの概要ページ

    始めにアプリを開いたり新規作成したりするとエディタペインに以下のような、アプリの概要を示すページを表示します:

    左側のツリーのルートをクリックすることで、いつでもアプリの概要ページに戻れます。

    マニフェストの検証

    WebIDE は自動的に、マニフェストでよく発生する問題点のいくつかを確認します。問題点を発見するとアプリの概要で、アプリが不正な状態であることと問題点の説明を表示します:

    manifest.webapp ファイルもプロジェクトエディタで編集できます。

    ソースエディタ

    WebIDE では CodeMirror ソースエディタを使用しています。

    ソースエディタのショートカット

    以下の表は、ソースエディタのデフォルトのショートカットの一覧です。

    Vim や Emacs のキーバインドも選択できます。これらを選択するには、about:config で設定項目 devtools.editor.keymap を選択して、"vim" または "emacs" を設定してください。この設定により、ソースエディタを使用するすべての開発ツールで選択したキーバインドを使用します。設定を反映させるには、エディタを開き直さなければなりません。

    コード補完

    CSS やJavaScript を編集する際は、エディタがオートコンプリートの候補を提示します。CSS のオートコンプリートは常に有効です:

    JavaScript のオートコンプリート候補は Control + Space を押下すると表示されます:

    インラインドキュメント

    エディタは、JavaScript のドキュメントをインラインで表示します。Shift + Space を押下すると、カーソルが置かれているシンボルに関するドキュメントをポップアップ表示します:

    ポップアップ内にある [docs] のリンクをクリックすると、そのシンボルに関する MDN のドキュメントを開きます。

    ファイルの保存

    ファイルに施した変更点は、保存しなければ反映されません。未保存の変更点があるファイルはツリービューでファイル名の隣にアスタリスクが表示されており、メニューを使用するか Control+S (Mac OS X では Command+S) を押下して保存できます。

    プロジェクトの削除

    アプリの概要ページで "Remove Project" をクリックすると、WebIDE からアプリが削除されます。

    アプリの実行とデバッグ

    アプリを実行する準備ができたら、"Select Runtime" ドロップダウンメニューで実行環境を選択しなければなりません。使用できる実行環境がない場合は、実行環境のセットアップで追加方法を見つけてください。

    WebIDE のツールバーの中央にある "play" が使用可能になります。このボタンをクリックすると、選択した実行環境にアプリをインストールして実行します:

    アプリのデバッグを行う場合は、"Pause" ボタンをクリックしてください。開発ツールのツールボックスが現れて、アプリに接続されます:

    どのツールを使用できるかは実行環境に依存しますが、少なくとも基本的なツールであるインスペクタ、コンソール、JavaScript デバッガ、スタイルエディタ、プロファイラ、スクラッチパッドを使用できます。Web ページと同様に、ツール内で施した変更点は直ちにアプリへ反映されますが、それは一時的です。逆に、エディタペインで施した変更点はディスクに直接保存できますが、アプリを再起動しなければ反映されません。

    認定アプリのデバッグ

    実行環境として Simulator を選択しているときは、アプリのドロップダウンメニューで自分のアプリだけでなく認定アプリを含む実行環境内の全アプリが表示され、またデバッグも可能です:


    一方、実機で認定アプリのデバッグを行うには以下の条件を満たさなければなりません:

    • デバイスで Firefox OS 1.2 以上の開発ビルドを実行していなければなりません
    • 認定アプリのデバッグを有効化していなければなりません

    認定アプリのデバッグを有効化するには実行環境に接続して、メニューで Runtime > Runtime Info を選択します。ここで "DevTools restricted privileges: yes" と表示されている場合は、認定アプリをデバッグできないことを意味します。デバイスが Rooted であるなら、"request higher privileges" をクリックすると認定アプリのデバッグが可能になります (Firefox OS が再起動します)。

    以上で、デバイス上のすべての認定アプリが WebIDE に表示されるようになるでしょう。

    トラブルシューティング

    WebIDE の作業においてトラブルが発生した場合は、トラブルシューティングのページをご覧ください。
